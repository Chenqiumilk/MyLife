# Portfolio Case Study Draft

## Project

Diary Focus: a cute diary and focus app concept.

## Role

Product design, prototyping, frontend implementation, iOS planning.

## Problem

Many diary apps are either too plain or too heavy. Many focus apps feel punitive or productivity-obsessed. Diary Focus explores a softer loop: record daily life, focus with a companion, and see effort reflected back in the diary.

## Users

- Students.
- Researchers.
- Reflective users.
- People who enjoy visual journaling.

## Product Decisions

- Calendar first, not a landing page.
- Mood icons replace past-day numbers to make memory visual.
- Timestamp entries live inside the diary rather than separate logs.
- Focus time returns to the diary as a daily accomplishment.
- Pet companion makes focus emotionally lighter.

## Prototype

The project includes a web preview for fast iteration on Windows and an early SwiftUI scaffold for the future iOS app.

## Engineering Notes

- Web prototype uses HTML, CSS, and JavaScript.
- Diary data is stored in browser local storage.
- Images are compressed before storage.
- Focus sessions accumulate by day.

## Next Steps

- Rebuild MVP in SwiftUI.
- Add local persistence.
- Test with users through TestFlight.
- Prepare App Store materials.

## What This Project Shows

- Product thinking.
- Interaction design.
- Iterative prototyping.
- Attention to privacy.
- Ability to bridge design and implementation.
